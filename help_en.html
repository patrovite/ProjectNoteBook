Create task

  • Syntaxe : t [@projects] [Px] [>x] [%x] [%e] Text
  • Description : Create a task item.
  • Parameters :
    • [@projects] : "@" followed by a project name. No space or tab characters in the project name.
      • Optional: yes
      • Default: If no project is declare the task will be global.
      • example: @project1
    • [Px] : Priority. "x" is the level of priority. Value between 0 and 2. 0:Normal. 1:Average. 2:High
      • Optional: yes
      • Default: 0 (Normal)
      • example: p0
    • [>x] : End date. The date format depend of the application settings.
      • Optional: yes
      • Default: No end date
      • examples:
        >12 : 12th of the current month of the current year
        >12/02 : 12/02 of the current year
        >12/02/18 : 12/02/2018
        >12/02/2018
        >W12 : Start of the week 12 (first worked day of the week of the current year)
        ><W12 : End of the week 12 (last worked day of the week of the current year)
        >W12/11 : Start of the week 12 (first worked day of the week of the current year)
        ><W12/18 : End of the week 12 (last worked day of the week of the current year)
        >W12/2018 : Start of the week 12 (first worked day of the week of the current year)
        ><W12/2018 : End of the week 12 (last worked day of the week of the current year)
    • [%x] : Progress percentage. Value between 0 and 100.
      • Optional: yes
      • Default: 0%
      • example: %50 (50%)
    • [%e] : Task finished. Set the progress percentage to 100%.
      • Optional: yes
      • Default: No modification
      • example: %e
    • Text : Task text.
      • Optional: no
  • Examples :
    • t @project1 p1 %25 write the application: This create a task for "project1" with a priority 1 (average), a progress of 25% and the text "write the application"
    • t p2 fix the problem: This create a "global" task with a priority of 2 (high), a progress of 0% (default) and the text "fix the problem"
    • t @prj1 @prj2 @prj3 fix the problem: This create a task for the project "prj1", "prj2" and "prj3" with a priority of 0 (normal, default), a progress of 0% (default) and the text "fix the problem"

Create note

  • Syntaxe : n [@projects] [Px] Text
  • Description : Create a note item.
  • Parameters :
    • [@projects] : "@" followed by a project name. No space or tab characters in the project name.
      • Optional: yes
      • Default: If no project is declare the task will be global.
      • example: @project1
    • [Px] : Priority. "x" is the level of priority. Value between 0 and 2. 0:Normal. 1:Average. 2:High
      • Optional: yes
      • Default: 0 (Normal)
      • example: p0
    • Text : Note text.
      • Optional: no
  • Examples :
    • n @project1 p1 This is a note: This create a note for "project1" with a priority 1 (average) and the text "This is a note"
    • n The sky is blue: This create a "global" note with a priority of 0 and the text "The sky is blue"
    • n @prj1 @prj2 @prj3 The color is blue: This create a note for the projects "prj1", "prj2" and "prj3" with a priority of 0 (normal, default) and the text "The color is blue"

Edit item

  • Syntaxe : e num
  • Description : Copy the content of item "num" in the command line in order to be edited. Once the edition is finished press ENTER. The syntaxe is the same as the edited item (note or task)
  • Parameters :
    • num : Item number.
      • Optional: no
      • Default:
      • example: e 5 (Edit item number 5)
  • Examples :
    • e 5: This command display the content of the item 5. Edit the content and press ENTER to update the item

Edit project

  • Syntaxe : e @project new_project_name
  • Description : Rename a project
  • Parameters :
    • @project : Project name starting with "@"
      • Optional: no
      • Default:
    • new_project_name : New project name
      • Optional: no
  • Examples :
    • e @project1 project2: Rename "project1" to "project2". This will update all items containing "project1"

Delete item

  • Syntaxe : d num
  • Description : delete an item
  • Parameters :
    • num : Item number to delete
      • Optional: no
  • Examples :
    • d 5: Delete the item number 5

Delete project

  • Syntaxe : d @project
  • Description : delete a project and all its items
  • Parameters :
    • num : Item number to delete
      • Optional: no
  • Examples :
    • d @project1: Delete all the items where the project name is "project1"

Copy item

  • Syntaxe : c num [@project]
  • Description : Copy an item in project. If no project in mentioned then item is copied in the same project.
  • Parameters :
    • num : Item number to copy
      • Optional: no
    • @project : Project name starting with "@"
      • Optional: no
      • Default: Same project
  • Examples :
    • c 5 @project1: Copy item number 5 to the project "project1"
    • c 5: Copy item number 5 in the same project as item 5

Convert task<->note

  • Syntaxe : x num n/t
  • Description : Convert an item as note or task.
  • Parameters :
    • num : Item number to convert
      • Optional: no
    • n or t : "n" convert item as a note. "t" convert item as a "task".
      • Optional: no
  • Examples :
    • x 5 t: Convert item number 5 to task. If it's already a task then no change is done.
    • x 5 n: Convert item number 5 to note. If it's already a note then no change is done.

Filter projects

  • Syntaxe : f [+/-][@Projects]
  • Description : Set the project filter.
  • Parameters :
    • [+/-][@Projects] : "+" or no sign, add a project to the display filter. "-" remove a project to the display filter Item. In order to filter global item use "global" as project name.
      • Optional: no
  • Examples :
    • f @prj1: Add project "prj1" to the display filter. Items from "prj1" are displayed.
    • f +@prj1: Add project "prj1" to the display filter. Items from "prj1" are displayed.
    • f @prj1 @prj2: Add projects "prj1" and "prj2" to the display filter. Items from "prj1" and "prj2" are displayed.
    • f -@prj1: Remove project "prj1" from the display filter. Items from "prj1" are hided.
    • f -@prj1 -@prj2: Remove projects "prj1" and "prj2" from the display filter. Items from "prj1" and "prj2" are hided.
    • f +@prj1 -@prj2: Add project "prj1" to the display filter. Remove "prj2" from the display filter. Items from "prj1" are displayed and items from "prj2" are hided.
    • f -@global: Hide the global items from the display filter.
