Création d'une tâche

  • Syntaxe : t [@projects] [Px] [>x] [%x] [%e] Texte
  • Description : Création d'un élément tâche.
  • Paramètres :
    • [@projects] : "@" suivi du nom du projet. Pas d'espace ou de tabulation dans le nom du projet.
      • Optionnel: oui
      • Défaut: Si aucun projet est déclaré alors la tâche sera globale.
      • Exemple: @project1
    • [Px] : Priorité. "x" est le niveau de priorité. Valeur entre 0 et 2. 0:Normale. 1:Moyenne. 2:Haute
      • Optionnel: oui
      • Défaut: 0 (Normal)
      • Exemple: p0
    • [>x] : Date de fin. Le format de la date dépend des paramètres de l'application.
      • Optionnel: oui
      • Défaut: Pas de date de fin
      • Exemples:
        >12 : 12 du mois courant de l'année courante
        >12/02 : 12/02 de l'année courante
        >12/02/18 : 12/02/2018
        >12/02/2018
        >W12 : Début de la semaine 12 (Premier jour travaillé de la semaine de l'année courante)
        ><W12 : Fin de la semaine 12 (Dernier jour travaillé de la semaine de l'année courante)
        >W12/11 : Début de la semaine 12 (Premier jour travaillé de la semaine de l'année courante)
        ><W12/18 : Fin de la semaine 12 (Dernier jour travaillé de la semaine de l'année courante)
        >W12/2018 : Début de la semaine 12 (Premier jour travaillé de la semaine de l'année courante)
        ><W12/2018 : Fin de la semaine 12 (Dernier jour travaillé de la semaine de l'année courante)
    • [%x] : Pourcentage d'avancement. Valeur entre 0 et 100.
      • Optionnel: oui
      • Défaut: 0%
      • Exemple: %50 (50%)
    • [%e] : Tâche terminée. Met le pourcentage d'avancement à 100%.
      • Optionnel: oui
      • Défaut: Pas de modification
      • Exemple: %e
    • Texte : Texte de la tâche.
      • Optionnel: non
  • Exemples :
    • t @project1 p1 %25 écrire l'application: Cela crée une tâche pour "project1" avec une priorité 1 (moyenne), un avancement de 25% et un texte "écrire l'application"
    • t p2 régler le problème: Cela crée une tâche "global" avec une priorité de 2 (haute), un avancement de 0% (Défaut) et un texte "régler le problème"
    • t @prj1 @prj2 @prj3 régler le problem: Cela crée une tâche pour les projets "prj1", "prj2" et "prj3" avec une priorité de 0 (normale, défaut), un avancement de 0% (Défaut) et un texte "régler le problème"

Création d'une note

  • Syntaxe : n [@projects] [Px] Texte
  • Description : Crée une élément note.
  • Paramètres :
    • [@projects] : "@" suivi du nom du projet. Pas d'espace ou de tabulation dans le nom du projet.
      • Optionnel: oui
      • Défaut: Si aucun projet est déclaré alors la tâche sera globale.
      • Exemple: @project1
    • [Px] : Priorité. "x" est le niveau de priorité. Valeur de 0 à 2. 0:Normale. 1:Moyenne. 2:Haute
      • Optionnel: oui
      • Défaut: 0 (Normal)
      • Exemple: p0
    • Texte : Texte de la note.
      • Optionnel: non
  • Exemples :
    • n @project1 p1 Ceci est une note: Cela crée une note pour "project1" avec une priorité de 1 (moyenne) et le texte "Ceci est une note"
    • n Le ciel est bleu: Cela crée une note "global" avec une priorité de 0 et une texte "Le ciel est bleu"
    • n @prj1 @prj2 @prj3 La couleur est bleu: Cela crée une note pour les projets "prj1", "prj2" et "prj3" avec une priorité de 0 (normale, Défaut) et le texte "La couleur est bleu"

Edition d'un élément

  • Syntaxe : e num
  • Description : Copie le contenu de l'élément "num" dans la ligne de commande afin d'être éditée. Une fois éditée, appuyez sur ENTREE pour mettre à jour l'élément. La syntaxe est la même que l'élément édité (note ou tâche)
  • Paramètres :
    • num : Numéro de l'élément.
      • Optionnel: non
  • Exemples :
    • e 5: Cette commande affiche le contenu de l'élément 5. Editez le contenu et appuyez sur ENTREE pour mettre à jour l'élément

      • Défaut: Pas de date de fin

        >W12 : Début de la semaine 12 (Premier jour travaillé de la semaine de l'année courante)

Editer un projet

  • Syntaxe : e @project nouveau_nom_projet
  • Description : Renome un projet
  • Paramètres :
    • @project : Nom du projet débutant par "@"
      • Optionnel: non
      • Défaut:
    • nouveau_nom_projet : Nouveau nom de projet
      • Optionnel: non
  • Exemples :
    • e @project1 project2: Renome "project1" en "project2". Cela va mettre à jour tous les éléments contenant "project1"

Effacer un projet

  • Syntaxe : d @project
  • Description : efface un projet et tout ses éléments
  • Paramètres :
    • @project : Nom du projet à effacer débutant par "@"
      • Optionnel: non
  • Exemples :
    • d @project1: Efface tous les éléments où le nom de projet est "project1"

Copier un élément

  • Syntaxe : c num [@project]
  • Description : Copies un élément dans un projet. Si aucun projet n'est spécifié alors la copie est fait dans le même projet.
  • Paramètres :
    • num : Numéro de l'élément à copier
      • Optionnel: non
    • @project : Nom du projet débutant par "@"
      • Optionnel: non
      • Défaut: Même projet
  • Exemples :
    • c 5 @project1: Copies l'élément numéro 5 dans le projet "project1"
    • c 5: Copie l'élément numéro 5 dans le même projet

Convertion d'une tâche<->note

  • Syntaxe : x num n/t
  • Description : Converti un élément comme note ou tâche.
  • Paramètres :
    • num : Numéro de l'élément à convertir
      • Optionnel: non
    • n or t : "n" converti un élément en note. "t" converti un élément en tâche.
      • Optionnel: non
  • Exemples :
    • x 5 t: Converti un élément numéro 5 en tâche. Si l'élément est déjà une tâche alors aucun changement est réalisé.
    • x 5 n: Converti un élément numéro 5 en note. Si l'élément est déjà une note alors aucun changement est réalisé.

Filtrer les projets

  • Syntaxe : f [+/-][@Projects]
  • Description : Adjuste le filtre d'affichage.
  • Paramètres :
    • [+/-][@Projects] : "+" ou aucun signe, ajoute un projet au filtre d'affichage. "-" supprime un projet au filtre d'affichage. Pour filter les éléments globaux utilisez "global" comme nom de projet.
      • Optionnel: non
  • Exemples :
    • f @prj1: Ajoute le projet "prj1" au filtre d'affichage. Les éléments de "prj1" sont affichés.
    • f +@prj1: Ajoute le projet "prj1" au filtre d'affichage. Les éléments de "prj1" sont affichés.
    • f @prj1 @prj2: Ajoute les projets "prj1" et "prj2" au filtre d'affichage. Les éléments de "prj1" et "prj2" sont affichés.
    • f -@prj1: Supprime le projet "prj1" du filtre d'affichage. Les éléments de "prj1" sont cachés.
    • f -@prj1 -@prj2: Supprime les projets "prj1" et "prj2" du filtre d'affichage. Les éléments de "prj1" et "prj2" sont cachés.
    • f +@prj1 -@prj2: Ajoute le projet "prj1" au filtre d'affichage. Supprime "prj2" du filtre d'affichage. Les éléments de "prj1" sont affichés et les éléments de "prj2" sont cachés.
    • f -@global: Supprime les éléments globaux du filtre d'affichage.
